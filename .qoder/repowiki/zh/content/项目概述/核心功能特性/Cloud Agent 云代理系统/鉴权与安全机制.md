## 核心组件

### 设备令牌管理

设备令牌采用 JWT 格式，包含标准声明（iss、sub、exp、iat、nbf）和扩展字段（版本号 v、权限信息 r）。令牌验证通过以下函数实现：

- `isTokenValid(token: string): boolean` - 基础有效性检查
- `isTokenExpired(token: string, bufferSeconds?: number): boolean` - 过期时间检查（可设置缓冲期）
- `getTokenExpirationDate(token: string): Date | null` - 获取过期时间


### Cloud Agent 客户端安全

Cloud Agent 客户端负责与云端服务进行安全通信，使用双重认证机制：

- 设备令牌（X-Device-Token）：用于设备身份识别
- API 密钥（X-API-Key）：用于服务端访问控制

```mermaid
sequenceDiagram
participant Client as "CloudAgentClient"
participant Server as "Cloud Agent 服务器"
Client->>Server : GET /health (携带 X-Device-Token)
Server-->>Client : 200 OK 或 401 Unauthorized
alt 认证成功
Client->>Server : POST /v1/run (携带 X-Device-Token + X-API-Key)
Server-->>Client : JSON 响应 (ok, tokens_in, tokens_out, cost)
else 认证失败
Server-->>Client : 401 Unauthorized
Client-->>Client : 提供 API Key 配置提示
end
```

## 详细组件分析

### 设备令牌验证机制

设备令牌验证是 Cloud Agent 安全体系的核心，采用以下策略：

#### 令牌结构解析
- 支持标准 JWT 结构（头部.载荷.签名）
- 载荷部分使用 base64url 编码
- 自动填充填充字符以确保解码正确性

#### 时间窗口控制
- 默认缓冲期：24小时，防止时钟偏差导致的误判
- 可配置缓冲期，适应不同业务场景
- 精确到秒的时间戳比较

```mermaid
flowchart TD
TokenInput["输入 JWT 令牌"] --> SplitParts["分割为三部分"]
SplitParts --> CheckCount{"是否恰好3部分?"}
CheckCount --> |否| ReturnNull["返回 null"]
CheckCount --> |是| DecodePayload["base64url 解码载荷"]
DecodePayload --> ParseJSON["解析 JSON"]
ParseJSON --> CheckExp{"存在 exp 字段?"}
CheckExp --> |否| ReturnTrue["返回 true (视为过期)"]
CheckExp --> |是| CalcBuffer["计算缓冲时间"]
CalcBuffer --> CompareTime["比较当前时间与过期时间"]
CompareTime --> Expired{"是否已过期?"}
Expired --> |是| ReturnTrue
Expired --> |否| ReturnFalse["返回 false (有效)"]
```

### Cloud Agent 客户端安全实现

#### 请求头安全策略
客户端在每个请求中自动添加必要的安全头信息：

| 头部名称 | 用途 | 必填性 | 示例值 |
|---------|------|--------|--------|
| Content-Type | 指定请求体格式 | 必需 | application/json |
| X-Device-Token | 设备身份标识 | 必需 | 从用户令牌派生 |
| X-API-Key | 服务端访问密钥 | 可选 | 仅在配置时发送 |

当服务器返回 401 未授权错误时，客户端提供智能诊断提示：

## 故障排除指南

### 常见认证问题

#### 401 未授权错误
当服务器返回 401 错误时，客户端会提供具体的配置指导：

**可能原因及解决方案：**
- API Key 配置错误：检查 VS Code 设置中的 `njust-ai.cloudAgent.apiKey`
- 环境变量未设置：确认 `CLOUD_AGENT_MOCK_API_KEY` 或 `NJUST_CLOUD_AGENT_API_KEY`
- 令牌过期：执行 `njust-ai auth login` 重新认证

#### 令牌验证失败
**排查步骤：**
1. 检查令牌格式是否为有效的 JWT
2. 验证载荷部分的 JSON 格式
3. 确认时间戳字段的有效性
4. 检查缓冲期设置是否合理

#### 网络连接问题
**诊断方法：**
- 验证服务器 URL 的可达性
- 检查防火墙和代理设置
- 确认 HTTPS 证书的有效性
- 测试基本的网络连通性

## 结论

Cloud Agent 的鉴权与安全机制通过多层次的设计实现了全面的安全防护。设备令牌验证确保了客户端身份的真实性，Cloud Agent 客户端的安全通信保障了数据传输的机密性和完整性，而完善的 CLI 认证工具链则提供了便捷的用户体验。
